# F-006 手机端录入 — Feature Scope

**创建方式**: adopt-scan 自动提取  
**最后更新**: 2026-06-02

---

## 目标

让现场操作人员用手机直接拍照录入，数据实时同步到桌面端，解决电脑不在手边时的录入需求。

## 边界

**包含**：
- 独立移动端页面（mobile.html），针对小屏优化
- 拍照（调用摄像头）或相册选图
- 单张图片 OCR 识别（共享 ocr-parser.js）
- OCR 原始文本展示与手动重新解析
- 面料信息表单（B/C/D/E/F/G/H/I 列字段）
- C 列编号自动补全（输入5位数 → HLWG/HLFG）
- G 列输入后 H 列实时计算（英寸）
- 操作人姓名记忆（localStorage 持久化）
- 提交后 POST 到 `tables/fabric_records` 并实时同步到桌面
- 一键清空表单
- 图片压缩缩略图（最大120px）

**不包含**：
- 查看已有数据列表
- 历史记录查看
- Excel 导出
- 多张图片批量识别

## 核心能力

1. 拍照或选图后可立即触发 OCR 识别
2. OCR 完成后自动填充各字段，字段可手动修改
3. 点击「保存并同步」提交数据到后端，桌面端下次同步时自动显示

## 涉及端

| 端 | 覆盖能力 |
| -- | -------- |
| 手机端 | 完整功能（独立页面） |
| 桌面端 | 不涉及（桌面端通过 F-003 同步接收手机端数据） |

## 对外 Facade

无

## 共享 Domain

- **ocr-parser.js**：与 F-001 共用 OCR 解析引擎（`parseOCRFull`、`resolveHLCode`、`calcInchDisplay`）
- **fabric_records 数据表**：与 F-002 共用数据表

## 关键实现位置

| 功能 | 文件 | 说明 |
| ---- | ---- | ---- |
| 全部逻辑 | mobile.html（内联 script） | 独立实现，不依赖 app.js/sync.js/history.js |
