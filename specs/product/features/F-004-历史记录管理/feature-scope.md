# F-004 历史记录管理 — Feature Scope

**创建方式**: adopt-scan 自动提取  
**最后更新**: 2026-06-02

---

## 目标

提供按批次查阅历史录入记录的能力，方便追溯特定时间段的录入内容，并支持将历史数据重新载入进行复核或修改。

## 边界

**包含**：
- 历史面板（右侧滑入侧边栏）
- 批次列表（时间、操作人、记录条数）
- 批次展开（图片缩略图网格 + 数据摘要表：编号/成分/克重/工厂批号）
- 将历史批次载入当前表格（清空当前数据后替换）
- Session 元信息保存（每次 OCR 批量识别完成后自动记录）

**不包含**：
- 历史批次删除
- 历史数据导出
- 跨批次对比
- 批次合并

## 核心能力

1. 点击「历史」按钮打开历史面板，从 `fabric_sessions` 表加载批次列表
2. 展开某批次查看图片缩略图和数据预览
3. 点击「载入此批次」将该批次数据替换当前表格内容
4. OCR 批量识别完成时自动保存 session 元信息（时间、操作人、行数）

## 涉及端

| 端 | 覆盖能力 |
| -- | -------- |
| 桌面端 | 完整功能 |
| 手机端 | 不涉及 |

## 对外 Facade

无

## 共享 Domain

- **fabric_sessions 数据表**：历史模块独占，记录每次录入批次的元信息
- **fabric_records 数据表**：历史模块通过 session_id 筛选读取

## 关键实现位置

| 功能 | 文件 | 函数 |
| ---- | ---- | ---- |
| 打开历史面板 | js/history.js | `openHistory()` |
| 加载批次列表 | js/history.js | `loadHistorySessions()` |
| 展开批次详情 | js/history.js | `toggleHistDetail()` |
| 载入历史批次 | js/history.js | `loadSessionToTable()` |
| 保存 session 元信息 | js/history.js | `saveSessionMeta()` |
