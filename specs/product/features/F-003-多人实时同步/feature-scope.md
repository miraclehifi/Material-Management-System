# F-003 多人实时同步 — Feature Scope

**创建方式**: adopt-scan 自动提取  
**最后更新**: 2026-06-02

---

## 目标

多人同时录入时，各端数据自动合并，用户无需手动刷新即可看到其他人新增的记录。

## 边界

**包含**：
- 定时轮询（默认每15秒，可配置）
- 增量同步（只拉取本地不存在的新行，避免覆盖用户正在编辑的数据）
- 编辑冲突规避（当前正在编辑的行跳过同步更新）
- 在线人数检测（localStorage 心跳，10秒一次，30秒超时清除）
- 同步状态指示器（表头旋转图标）
- 同步日志追加到 OCR 日志区

**不包含**：
- WebSocket 实时推送
- 操作锁定（不阻止多人同时编辑同一行）
- 冲突合并（后写覆盖）

## 核心能力

1. 页面加载时启动同步定时器，按配置间隔轮询
2. 发现其他用户新增的行时自动追加到本地表格
3. 发现其他用户修改的行时，若本地未聚焦该行则静默更新
4. 通过 localStorage 心跳统计当前在线的标签页/浏览器数量

## 涉及端

| 端 | 覆盖能力 |
| -- | -------- |
| 桌面端 | 完整功能 |
| 手机端 | 不涉及（手机端提交后即完成，依赖 F-002 的加载机制） |

## 对外 Facade

无

## 共享 Domain

- **fabric_records 数据表**：与 F-002 共享，同步模块直接操作 `tableRows` 全局状态

## 关键实现位置

| 功能 | 文件 | 函数 |
| ---- | ---- | ---- |
| 同步启动 | js/sync.js | `startSync()` |
| 主同步逻辑 | js/sync.js | `doSync()` |
| 在线心跳 | js/sync.js | `doHeartbeat()` |
